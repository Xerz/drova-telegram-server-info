# Telegram UX Current Spec

## Общие правила

- Бот работает через `python-telegram-bot` long polling.
- Все основные сообщения отправляются как HTML (`ParseMode.HTML`); при ошибке парсинга бот повторяет отправку plain text.
- Если для чата не сохранен `X-Auth-Token`, защищенные команды отвечают `setup me first`.
- Inline-кнопки обновляют исходное сообщение через `edit_message_text`; ошибка `message is not modified` считается нефатальной.
- Любой обычный текст без команды получает ответ `Sorry, I don't understand that message.`

## Команды

| Команда | Поведение |
| --- | --- |
| `/start` | Показывает справку со списком базовых команд и предупреждением про токены. |
| `/token <token>` | Проверяет токен через `GET /accounting/myaccount`; при успехе сохраняет токен и `uuid` аккаунта, обновляет кэш продуктов и имен станций. |
| `/removeToken` | Удаляет токен из локального хранилища; подтверждение отправляется только если токен был найден. |
| `/station` | Загружает станции, сохраняет их имена и показывает inline-кнопки выбора станции плюс `all`. |
| `/station <station_id>` | Сохраняет выбранный station id без live-валидации. Значение `-` означает все станции. |
| `/limit <N>` | Сохраняет лимит сессий как `int`; текущий код не валидирует диапазон и текст ошибки. |
| `/sessions` | Показывает последние `limit` сессий по выбранной станции или по всем станциям. |
| `/sessions short` | То же, но выводит только сессии длиннее 5 минут. |
| `/current` | Показывает список станций с последней сессией каждой станции и inline-кнопками обновления/publish. |
| `/disabled` | Показывает продукты, где `enabled`, `published` или `available` не true. |
| `/stationsInfo` | Показывает станции, город и внешние/внутренние endpoint addresses. |
| `/dumpall` | Отправляет CSV-файл по каждой станции с доступными сессиями. |
| `/dumpOnefile` | Отправляет один XLSX со всеми сессиями. |
| `/dumpStationsProducts` | Отправляет XLSX-матрицу продуктов по станциям и состояниям. |
| `/dumpStationsProductsWithTime` | Отправляет XLSX-матрицу с суммарным временем по продуктам. |
| `/dumpStationsProductsMonth` | То же, но только за последние 30 дней. |

## Callback-сценарии

- `update_sessions` и `update_sessions_short` пересобирают `/sessions` и редактируют исходное сообщение.
- `set_server_id_{uuid}` сохраняет выбранную станцию; `set_server_id_-` выбирает все станции.
- `current:update` пересобирает `/current`.
- `current:publish:show` и `current:publish:hide` показывают/скрывают кнопки публикации.
- `current:publish:toggle:{server_uuid}` меняет `published` у станции и затем обновляет `/current`.
- `update_disabled` и `update_stationsinfo` обновляют соответствующие сообщения.

## Формат сообщений

- `/sessions` группирует записи по дате `created_on`, выводит игру, станцию, IP/client suffix, GeoIP city/ASN, время старта-финиша, длительность, feedback, billing type и status.
- `/current` сортирует станции по имени и нумерует их. Для каждой станции показывает форматированное имя, последнюю игру, trial-marker, GeoIP city, расстояние и длительность.
- `/stationsInfo` разделяет endpoints на внешние и RFC1918-внутренние адреса.
- `/disabled` возвращает `all products fine`, если проблемных продуктов нет.

## Текущие UX-долги

- Команды и ответы смешивают русский и английский.
- `/station` строит все кнопки в одной строке, что плохо масштабируется.
- `/limit` и `/station <id>` принимают невалидные значения без нормальной проверки.
- Ошибки API часто схлопываются до `Error` или `Error: <status>` без подсказки пользователю.
- Publish-toggle доступен из `/current`, но нет дополнительного подтверждения опасного действия.
