# Product Spec

## Пользователи

- Основной пользователь: владелец Drova-станций, который смотрит актуальные сессии, состояние станций, проблемные продукты и выгрузки.
- Вторичный пользователь: разработчик/оператор бота, который проверяет API-контракты, деплой и live checks.

## Основные сценарии

- Подключить Drova account через proxy token.
- Выбрать станцию или режим "все станции".
- Посмотреть последние сессии с понятной группировкой, фильтром коротких сессий и кнопкой обновления.
- Посмотреть текущий статус станций и последние сессии по каждой.
- Посмотреть агрегированную статистику использования за сегодня, неделю и месяц.
- Временно скрыть или опубликовать тестовую/рабочую станцию через подтверждаемый publish-flow.
- Найти отключенные или недоступные продукты.
- Посмотреть endpoints станций, разделенные на внешние и внутренние.
- Посмотреть игры выбранной станции, параметры запуска, скрыть/открыть игру на выбранной станции или скрыть игру на всех станциях.
- Включить или выключить полный доступ и обновления на выбранной станции через command-confirm flow.
- По явной команде посмотреть исходник описания выбранной станции.
- Обновить описание выбранной станции через preview/apply с защитой от устаревшего исходника.
- Скачать CSV/XLSX выгрузки по сессиям и продуктам.

## Команды V2

| Команда | Назначение |
| --- | --- |
| `/start` | Онбординг, статус подключения, список основных действий. |
| `/help` | Полная справка по командам и безопасности токена. |
| `/token <token>` | Подключить или заменить Drova proxy token. |
| `/logout` | Удалить токен, user id, выбранную станцию и приватные настройки чата. |
| `/station` | Открыть выбор станции через paginated inline UI. |
| `/station_all` | Выбрать все станции. |
| `/limit <N>` | Установить лимит последних сессий, допустимо `1..100`. |
| `/sessions` | Последние сессии по выбранной станции или всем станциям. |
| `/sessions_short` | Последние сессии дольше 5 минут. |
| `/current` | Dashboard состояния всех станций. |
| `/account` | Баланс минут, открытые выплаты и последние операции с минутами. |
| `/usage` | Статистика использования за сегодня, неделю и месяц. |
| `/disabled` | Проблемные продукты по станциям. |
| `/stations` | Список станций и endpoints. |
| `/games` | Постраничный выбор игры на выбранной станции, по названиям и состоянию. |
| `/desktop_on` | Включить полный доступ на выбранной станции после подтверждения. |
| `/desktop_off` | Выключить полный доступ на выбранной станции после подтверждения. |
| `/updates_on` | Включить обновления на выбранной станции после подтверждения. |
| `/updates_off` | Выключить обновления на выбранной станции после подтверждения. |
| `/server_source` | Показать исходник описания выбранной станции. |
| `/server_description <text>` | Подготовить обновление описания выбранной станции. |
| `/promocode <minutes>` | Выпустить один prepaid-промокод на указанное целое число минут. |
| `/promocodes` | Показать неактивированные prepaid-промокоды. |
| `/export_sessions` | Выгрузить сессии одним XLSX-файлом. |
| `/export_sessions_csv` | Выгрузить сессии CSV-файлами по станциям. |
| `/export_products` | Выгрузить XLSX-матрицу состояния продуктов. |
| `/export_product_time` | Выгрузить XLSX времени использования продуктов. |

Backward-compatible aliases должны поддерживаться в V2: `/station all`, `/sessions short`, `/export sessions`, `/export products`, `/export product-time`, `/removeToken`, `/stationsInfo`, `/dumpall`, `/dumpOnefile`, `/dumpStationsProducts`, `/dumpStationsProductsWithTime`, `/dumpStationsProductsMonth`.

Technical game-management aliases may stay accepted for automation/debugging but are
not primary UX: `/game <product_id>`, `/game_hide <product_id>`,
`/game_show <product_id>`, `/game_hide_all <product_id>`.

## UX Defaults

- Язык интерфейса: русский.
- Время отображать в timezone процесса, по умолчанию `Asia/Yekaterinburg`, но хранить timestamp в UTC.
- Лимит сессий по умолчанию: `5`.
- Short mode: показывать только сессии строго дольше 5 минут.
- Ошибки должны быть пользовательскими, без raw tracebacks и без token/IP leakage.

## Permission And Safety

- Все команды кроме `/start`, `/help`, `/token` требуют сохраненного Drova token.
- Publish-toggle всегда требует inline confirmation.
- Live write checks и specing write operations разрешены только для `TEST_STATION_UUID`.
- В production UX publish-toggle разрешен для любой станции пользователя, но действие подтверждается отдельной кнопкой и показывает новое состояние.
